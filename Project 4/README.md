# Pet Cafe Database System

## Overview
Designed and implemented a fully normalized relational database for a hypothetical Pet Cafe organization using Oracle SQL in a shell-scripted environment. The system accurately tracks members, pets, daily activities, and transactions.

## Goals
- Design a database schema from scratch that adheres to Boyce-Codd Normal Form (BCNF)
- Implement the schema in Oracle with proper relational constraints
- Automate setup and teardown via shell scripts

## Tools & Technologies
- **Languages:** SQL, Java, Shell Script
- **Database:** Oracle
- **Environment:** Terminal / Shell

## Schema Highlights
- **15 tables**, **15 entities**, **17 relationships**
- Primary keys and foreign keys enforced throughout
- Fully BCNF-compliant — no partial or transitive dependencies
- Models: members, pets, staff, activities, transactions, appointments

## Key Skills Demonstrated
- Relational database design and normalization
- Oracle SQL DDL and DML
- Shell scripting for database automation
- Entity-relationship modeling
